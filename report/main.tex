\begin{abstract}
This report presents the architectural design, algorithmic implementation, and rigorous empirical evaluation of \textbf{DevSeek}---a specialized, domain-specific (Vertical) Search Engine tailored for information technology (IT) programming tutorials, software architecture, and technical documentation. Designed to achieve technical excellence across five integrated modules, the system features: (1) Scalable automated data acquisition managing \textbf{520+ technical articles} across 15 specialized categories (Python, JavaScript, AI/ML, DevOps, System Design), stored synchronously in JSON, CSV, and SQLite formats; (2) Advanced Vietnamese Natural Language Processing combining \texttt{underthesea} compound word segmentation with an \textbf{Automated Synonym Expansion Map}, indexed via a positional Inverted Index tracking field-level term frequencies; (3) A \textbf{Dual Ranking Engine} supporting instantaneous switching between weighted \textbf{Multi-Field TF-IDF} ($\times 3.0$ boost for Title, $\times 2.5$ for Tags) and industry-standard \textbf{Okapi BM25F} ($k_1=1.5, b=0.75$); (4) A premium Flask Web Application built with dark-mode glassmorphism aesthetics, faceted filtering, dynamic sorting, and keyword highlighting using HTML \texttt{<mark>} tags; and (5) An automated comparative evaluation suite benchmarked against 20 curated queries with expert-labeled ground truth. Experimental results demonstrate outstanding performance: \textbf{Multi-Field TF-IDF achieves Mean P@10 = 0.9450 (94.5\%) and MAP = 0.6139}, while \textbf{Okapi BM25F achieves Mean P@10 = 0.8900 (89.0\%) and MAP = 0.5883}, with average query execution latencies of approximately $\sim 30.67$ ms.
\end{abstract}

\section{Introduction \& Domain Specificity}
% ==============================================================================

\subsection{What is a Vertical Search Engine?}
General-purpose web search engines such as Google and Bing index billions of web pages across all conceivable topics. However, when software engineers or computer science students search for technical tutorials, algorithmic explanations, or API documentation, general search engines often return noisy results mixed with e-commerce, generic tech journalism, or SEO spam.

A \textbf{Vertical Search Engine} addresses this limitation by restricting its scope to a specific domain. By focusing exclusively on \textbf{520+ curated IT programming resources}, DevSeek applies domain-specific tokenization rules (preserving critical programming symbols such as \texttt{C++}, \texttt{C\#}, \texttt{K8s}, \texttt{CI/CD}, and \texttt{Async/Await}) and implements field-weighted ranking algorithms optimized specifically for structured technical literature.

\subsection{Project Objectives \& Technical Targets}
The DevSeek engine is engineered to satisfy the highest academic and engineering standards across four primary dimensions:
\begin{itemize}
    \item \textbf{Scale \& Storage Synchronicity:} Automated collection, generation, and indexing of \textbf{520+ technical articles}, synchronously persisted across JSON, CSV, and relational SQLite (`devseek.db`) databases.
    \item \textbf{Domain-Aware Natural Language Processing:} Vietnamese compound term segmentation via \texttt{underthesea} integrated with an automated query expansion dictionary (\texttt{js -> javascript}, \texttt{csdl -> database}).
    \item \textbf{Dual Ranking Engine Architecture:} Simultaneous execution and real-time comparison of Multi-Field TF-IDF and Okapi BM25F ranking algorithms.
    \item \textbf{Quantitative Empirical Validation:} Automated, reproducible benchmarking measuring Precision@10 and Mean Average Precision (MAP) across 20 curated queries.
\end{itemize}

\section{Core Ranking Algorithms \& Mathematical Formulations}
% ==============================================================================

\subsection{Multi-Field TF-IDF (Title \& Tags Prioritization)}
The relevance score between a user query $Q$ and document $D$ is formulated using a field-weighted TF-IDF model:
\begin{equation}
\text{Score}_{\text{TFIDF}}(Q, D) = \sum_{t \in Q} \text{IDF}(t) \times \text{TF}_{\text{weighted}}(t, D)
\end{equation}
where smoothed Inverse Document Frequency is defined as $\text{IDF}(t) = \ln \left( \frac{N + 1}{\text{df}(t) + 1} \right) + 1.0$. To reflect structural document hierarchy, field weights are assigned as:
\begin{equation}
\text{WeightedTF}(t, D) = 3.0 \cdot \text{TF}_{\text{title}} + 2.5 \cdot \text{TF}_{\text{tags}} + 1.5 \cdot \text{TF}_{\text{summary}} + 1.0 \cdot \text{TF}_{\text{content}}
\end{equation}
Sublinear logarithmic scaling ($1 + \ln(\text{WeightedTF})$) is applied to prevent verbose documents from dominating term frequency counts.

\subsection{Okapi BM25F (Industry-Standard Field Normalized Ranking)}
To enhance ranking stability across documents with variable field lengths, DevSeek implements Okapi BM25F. Field term frequencies are normalized relative to average field lengths across the corpus:
\begin{equation}
\text{NormTF} = \sum_{f} w_f \cdot \frac{\text{TF}_f}{1 - b + b \cdot \left( \frac{\text{length}_f}{\text{avg\_length}_f} \right)}
\end{equation}
\begin{equation}
\text{Score}_{\text{BM25F}}(Q, D) = \sum_{t \in Q} \text{IDF}_{\text{BM25}}(t) \times \frac{\text{NormTF} \cdot (k_1 + 1)}{\text{NormTF} + k_1}
\end{equation}
We adopt standard information retrieval hyperparameters: $k_1 = 1.5$ (term saturation threshold) and $b = 0.75$ (length normalization penalty).

\subsection{Empirical Analysis \& Insights}
\begin{itemize}
    \item \textbf{Multi-Field TF-IDF Superiority in Precision@10 (94.5\% vs 89.0\%):} By heavily boosting exact matches in Title ($\times 3.0$) and Tags ($\times 2.5$), TF-IDF excels when queries contain exact technical keywords.

    \item \textbf{Real-Time Execution Latency:} Both ranking engines complete full positional evaluation across 520 articles in approximately \textbf{$\sim 29.25 - 30.67$ ms}, fully satisfying real-time web application latency targets.
\end{itemize}
